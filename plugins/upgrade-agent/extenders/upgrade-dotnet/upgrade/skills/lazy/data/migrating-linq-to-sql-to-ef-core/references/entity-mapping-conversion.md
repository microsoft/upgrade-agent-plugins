# Entity Mapping Conversion

## Attribute Mapping Table

| LINQ to SQL (`System.Data.Linq.Mapping`) | EF Core Equivalent | Notes |
|---|---|---|
| `[Table(Name="X")]` | `[Table("X")]` (`System.ComponentModel.DataAnnotations.Schema`) | Property `Name` → constructor param |
| `[Column(Name="X")]` | `[Column("X")]` | Namespace change only |
| `[Column(IsPrimaryKey=true)]` | `[Key]` | Different attribute |
| `[Column(IsDbGenerated=true)]` | `[DatabaseGenerated(DatabaseGeneratedOption.Identity)]` | Different attribute entirely |
| `[Column(IsVersion=true)]` | `[Timestamp]` or `[ConcurrencyCheck]` | See concurrency reference |
| `[Column(UpdateCheck=...)]` | No direct equivalent | See `concurrency-and-change-tracking.md` |
| `[Column(CanBeNull=true)]` | Nullable reference type or `.IsRequired(false)` | Different mechanism |
| `[Column(Storage="_field")]` | `.HasField("_field")` in Fluent API | No attribute equivalent |
| `[Column(AutoSync=...)]` | `[DatabaseGenerated]` + store-generated patterns | Different model |
| `[Column(DbType="Decimal(18,4)")]` | `[Precision(18, 4)]` (EF Core 6+) or `.HasPrecision(18, 4)` | **Critical**: EF Core defaults to `decimal(18,2)` — silent data truncation if not configured |
| `[Association(...)]` | Navigation property + Fluent API `HasOne`/`HasMany` | Completely different mechanism |
| `[InheritanceMapping(...)]` | `.HasDiscriminator()` in Fluent API | Attribute-per-subtype → fluent config |
| `[Function(Name="sp_X")]` | No attribute — use `FromSqlRaw` | See `stored-procedure-migration.md` |

## DBML Conversion Strategy

**Recommended: Scaffold first, then reconcile.**

1. Run `dotnet ef dbcontext scaffold "ConnectionString" Microsoft.EntityFrameworkCore.SqlServer -o Entities`
2. Compare scaffolded output with DBML definitions:
   - DBML stores both DB name and C# name — scaffolded names match DB, not custom C# names
   - Rename scaffolded entities/properties to match existing code expectations
3. Merge custom logic from partial classes extending DBML-generated entities
4. Apply Fluent API for non-convention mappings (decimal precision, column ordering, etc.)

## Handling Generated Designer Code

The `.designer.cs` file (e.g., `Northwind.designer.cs`) contains:
- Entity classes with `INotifyPropertyChanging`/`INotifyPropertyChanged`
- Partial methods: `OnXChanging()`/`OnXChanged()` for every property
- Backing field patterns (`_fieldName`)
- `EntitySet<T>`/`EntityRef<T>` navigation properties

**Do NOT edit this file. Do NOT rewrite EF Core entities into it.** Create new clean files instead:
- New `{Name}DbContext.cs` for the DbContext class
- New `Entities/{EntityName}.cs` files (or a single file for small models) for clean POCO entity classes
- Delete the `*.designer.cs` file entirely in the cleanup step (Step 9)

The `.designer.cs` filename implies auto-generation and will mislead future developers if it contains hand-written EF Core code.

## Partial Class Extensions

If the project has partial classes extending DBML-generated entities (e.g., `Customer.cs` alongside `Northwind.designer.cs`), these contain business logic that must be preserved. Steps:

1. Identify all partial class files (not the `.designer.cs`)
2. Move validation logic, computed properties, and methods to the new EF Core entity classes
3. Remove any references to `EntitySet<T>`, `EntityRef<T>`, or `System.Data.Linq.Mapping` attributes

## Namespace Collision Warning

Both LINQ to SQL and EF Core define `[Table]` and `[Column]` attributes:
- `System.Data.Linq.Mapping.TableAttribute`
- `System.ComponentModel.DataAnnotations.Schema.TableAttribute`

During migration, if both namespaces are in scope, use fully-qualified names:
```csharp
[System.ComponentModel.DataAnnotations.Schema.Table("Orders")]
public class Order { ... }
```

Remove the `System.Data.Linq.Mapping` using directive as soon as all attributes are converted.

## Serialization Mode

If the DBML has `Serialization Mode = Unidirectional`, entities have `[DataContract]`/`[DataMember]` attributes for WCF. If the project shares these entities with WCF service contracts, the migration touches both data access and service layer — plan accordingly.
